# Artifact Style

For handoff, rollout, status, and similar operator artifacts:

- Prefer compact markdown.
- Use short section headings.
- Preserve latest confirmed facts, not stale baseline values.
- Keep the artifact practical for the next operator.
- When listing open work, keep it concise.
- When listing risks, include grounded risks only.
- Do not rewrite an artifact if the user asked only for inspection.
- If verification is requested, read the artifact back after writing.
- Avoid decorative prose, long introductions, tables, and unnecessary formatting flourishes.

For checkpoint and wrap-up responses in chat:

- Use flat factual bullets.
- Put the main answer early.
- Avoid headings unless explicitly requested.
- Avoid tables unless explicitly requested.
